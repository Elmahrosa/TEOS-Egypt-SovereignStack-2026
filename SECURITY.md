# Security Policy

## Reporting a Vulnerability
If you discover a vulnerability or security issue in any reference implementation, documentation, or example code associated with TEOS:

- Do not disclose publicly.
- Send a responsible disclosure report to: **[SECURITY_EMAIL]**
- Include: description, impact, reproduction steps, and suggested mitigations.

## Scope
This repository is documentation and reference architecture. Any deployed system should undergo:
- Independent security review
- Threat modeling
- Audit and controls verification
- Compliance validation per jurisdiction
